发表于 2020-09-05

最开始，你需要一个支持cuda的gpu，需要去英伟达官网查看。如果你的gpu不支持，那就只能用cpu来跑了。一般来说，gpu在训练深度神经网络的方面，能力是cpu的十几二十倍。如果支持，那么就开始下面的教程吧。

  • ubuntu安装软件大概有三种方式，
    1 通过ubuntu官方源，直接用apt-get install
    2 通过第三方源，添加ppa后，再用apt-get install
    3 通过软件公司提供的安装包或脚本。若是安装包会用到dpkg,若是安装脚本，直接执行即可，有时需要root用户权限。

树莓派安装opencv-python3最简单的方式

发表于 2020-09-05

由于的树莓派的处理器属于arm系列，安装一些包时，需要自己下载源码编译。但是由于树莓派的网络传输以及cpu的处理速度太慢，编译像opencv这种稍大型的程序时，动辄需要两三个小时。不仅如此，还很容易编译错误。

有一种方式，可以很轻松地完成python3的opencv库的安装。废话少说，直接给出答案，就是Berryconda 。Berryconda 是一款为树莓派定制基于 Conda Python 发行版的开源软件。可以使用它来实现类似 Miniconda 和 Anaconda 的功能。

我们知道，Anaconda里的包可能不是最新的，但是，当我们着急要使用python的某个包时，通过conda来安装是最好的方式。

安装Berryconda

安装很简单，按照github直接下载脚本执行，并按照提示，修改.bashrc就行，

目前Berryconda的opencv版本是3.3,直接conda install opencv，只需二十几分钟即可安装成功。如果你必须要使用高于这个版本的opencv，那只能通过编译安装了。

由于树莓派的处理能力有限，我们一般只会让它完成一些简单的图片和摄像头操作，没有必要安装最新的版本，能用就行。

tensorflow2.0 用卷积层报错

发表于 2020-09-05

报错信息

UnknownError: Failed to get convolution algorithm. This is probably because cuDNN failed to initialize, so try looking to see if a warning log message was printed above. [Op:Conv2D]

解决办法

具体原因是因为tensorflow默认会把显存一次性占完，导致后面用卷积层时无法初始化cuDNN，所以，我们只要把tf设置为按需申请显存就行了．

wps 目录，页眉和参考文献

发表于 2020-09-05

自动创建目录

  1. 从摘要开始一直到文章结束，选中之后选择段落，然后设置为正文文本。
  2. 对于摘要、章节名称、参考文献和致谢等等，选中后在段落设置为一级标题。
  3. 同样的，对于像1.1 研究背景和2.2 XXX这样有两个数字的小标题，选中后设置为二级标题。
  4. 接着使用同样的方法对三个数字的小标题设置三级标题，一般来说，目录只需要显示到三级标题。
  5. 在摘要前面插入一个空白页，在章节里点击目录页，选择相应的格式后就会插入之前所设置的目录了，最后需要按学校的要求修改字体和间距。

章节变化的页眉

通常，学校要求论文页数太多时采取双面打印格式，也就是每章节的开头需要在奇数页。如果不符合，需要在前一章插入空白页，这样打印出来之后阅读方便。

这时，奇数页的页眉需要设置为章节名称，偶数页的页眉需要设置为学校名称相关的名称。另外，页码也要在两侧显示。这里只介绍页眉的设置方法，页码不做介绍。
参考视频链接

1. 下一页分节符

  • 首先，为了达到章节之间相互独立，有一个前提两章之间有下一页分节符，其跟分页符类似，但效果更强，不仅会另起一页，还会起到隔离的效果。
  • 在章节名称前面插入页面布局-->分隔符-->下一页分节符 ，这时会另起一页，章节名称会出现在该页首行。

2. 奇偶页不同

  • 因为奇数页的页眉是章节名称，偶数页的页眉是学校名称相关的毕业论文，所以需要在页面设置-->版式-->页眉和页脚下面将奇偶页不同打勾。

    3. 同前节

  • 当编辑页眉时，如果是奇数页，需要将页眉和页脚下的同前节取消（颜色深时为选中，再点击一次颜色变浅为取消）。如果是偶数页，则无需取消同前节，因为偶数页的页眉是一样的。

参考文献

一般引用参考文献有两种方法，尾注法和编号法 。前一种插入时麻烦，但后来修改时方便，可以随意在中间插入和变换位置；后一种插入简单，但不便于修改，如果你要改变相对位置或添加删除某一个参考文献，会导致其他文献的编号错位。参考视频链接

为了方便，这里只介绍编号法的基本思路。

该方法将参考文献设置为item，然后在正文中通过交叉引用来插入文献的编号，然后设置为上脚标格式。 步骤为 reference --> Corss-reference -->Number item, Paragraph number(full context)，然后选择对应的文献，点击插入。该种方法适合于已经知道参考文献的顺序，且以后不会有太多的改变，否则，有可能牵一发而动全身。

c/c++ 常量和变量

发表于 2020-09-05

一，const 修饰的变量

  • int * p2 = &c_b;该句在使用gcc编译时，会警告warning: initialization discards ‘const’ qualifier from pointer target type [-Wdiscarded-qualifiers]。

  • 使用clang编译时，会警告 warning: initializing 'int *' with an expression of type 'const int *' discards qualifiers [-Wincompatible-pointer-types-discards-qualifiers]。

  • clang编译后运行的结果与gcc不同。如果使用clang编译，c_b的值没有改变，仍然是10；而使用gcc编译，该值则变为20。

从警告的内容可以看出，clang的提示要比gcc更加清晰明了；从执行结果可以看出，clang实现的const更加符合const的语义。

二，字符串常量和变量

  • char * c_p = "hello world";该语句定义的指针指向常量区地址，无法修改。

  • char v_p[] = "hello world";该语句会拷贝一份字符串到栈区，v_p是指向该地址的指针。

修改字符串常量是未定义行为，不同的编译器结果可能不同，本文使用的是clang和gcc

    四，其他说明

  • 编码规范是是公司规定的，我就懒得改了，如果看得不习惯自己改一改吧。

  • 在Windows下无法实现detach，这个我实在没找到解决方案，不过detach就是在linux下也要慎用。

  • 由于是用vs2010编译的，对c++11的支持还不足，无法使用thread_local关键字，所以采用了__declspec( thread )，该用法在vs2010中无法支持string这类复杂类型，所以只使用了一根指针指向了线程本身。这样在线程的回调函数中可以获得执行该函数的线程的信息，比如name，这在打日志时很有帮助。

  • 信号量的创建函数CreateSemaphore(NULL, 0, count, NULL)，中的count参数，与POSIX的含义不同，Windows下是指信号量的最大值，而POSIX中的是指信号量的初始值。

  • 在Thread的构造函数中，用到了信号量，这样做是为了保证在Thread对象构造完成时，里面的回调函数已经执行起来了。

  • 关于CSA锁(自旋锁)，这是我在网上搜到的解决方案。该锁会增加cpu的负载，适合对临界资源访问很快就结束的情况。基本原理就是，使用cpu空转，来避免进入休眠状态，进而提高效率。
