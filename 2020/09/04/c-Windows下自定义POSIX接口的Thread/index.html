c++ Windows下自定义POSIX接口的Thread

发表于 2020-09-04 更新于 2020-09-05

一，缘起

  • 入职后才知道公司的服务器是Windows ，目前使用的VS2010，正在向VS2017升级，无奈（#_#)。
  • 小组安排了一个定时器考核任务，需要用到多线程，由于自己之前都是在linux直接使用pthread（好吧，我还是强烈建议在linux或者Mac下学习c++），所以在Windows下使用多线程的经验为零。
  • 还好自己对多线程，以及线程之间同步（锁，信号量）机制有着还算入门的了解，同时在b站看了大佬sylar对pthread库的封装，于是就想着照虎画猫（大佬的是虎🐅，本人的是猫🐱），对Windows下的线程函数进行封装，实现类似POSIX的接口。

    四，其他说明

  • 编码规范是是公司规定的，我就懒得改了，如果看得不习惯自己改一改吧。

  • 在Windows下无法实现detach，这个我实在没找到解决方案，不过detach就是在linux下也要慎用。

  • 由于是用vs2010编译的，对c++11的支持还不足，无法使用thread_local关键字，所以采用了__declspec( thread )，该用法在vs2010中无法支持string这类复杂类型，所以只使用了一根指针指向了线程本身。这样在线程的回调函数中可以获得执行该函数的线程的信息，比如name，这在打日志时很有帮助。

  • 信号量的创建函数CreateSemaphore(NULL, 0, count, NULL)，中的count参数，与POSIX的含义不同，Windows下是指信号量的最大值，而POSIX中的是指信号量的初始值。

  • 在Thread的构造函数中，用到了信号量，这样做是为了保证在Thread对象构造完成时，里面的回调函数已经执行起来了。

  • 关于CSA锁(自旋锁)，这是我在网上搜到的解决方案。该锁会增加cpu的负载，适合对临界资源访问很快就结束的情况。基本原理就是，使用cpu空转，来避免进入休眠状态，进而提高效率。
